How to Free Up Disk Space on Windows 10 and 11

A practical walkthrough — from the quick wins to the deep cleaning strategies that actually move the needle.

🪟 Windows 10 / 11 ⏱ ~8 min read 📅 2024

Section 1

Why Windows drives fill up faster than you expect

A 500 GB SSD sounds like plenty of space until Windows starts warning you it's almost full. The reason is that Windows doesn't just store your files — it quietly accumulates its own pile of data in the background. Windows Update caches old patch files in case you need to roll back. Every browser session leaves cached images, scripts, and cookies. Applications write to temporary folders constantly and almost never clean up after themselves. Installer packages remain on disk long after the software they installed is running.

The result is that on a typical Windows 10 or 11 machine that has been running for a year or two, anywhere from 10 to 40 GB of disk space is occupied by files that can be safely deleted without affecting any installed software or your personal data. This article walks through each category from fastest to most impactful.

Section 2

Use Storage Sense to automate cleanup

Windows 10 and 11 both include a feature called Storage Sense that can automatically delete certain categories of temporary files on a schedule. It is disabled by default but takes about 60 seconds to configure.

1
Open Settings → System → Storage. You will see a breakdown of what is using space on each drive.
2
Click Storage Sense and toggle it on. Set it to run automatically — monthly is a reasonable default for most users.
3
Under "Delete files in my recycle bin if they have been there for over," set this to 30 days. Do the same for the Downloads folder if you're comfortable with that.
4
Click Run Storage Sense now to do an immediate cleanup pass. This typically recovers between 1 and 5 GB on a machine that hasn't been cleaned in a while.
Storage Sense targets temp files, Recycle Bin contents, and old Windows Update delivery optimization files. It does not touch your personal documents, photos, or installed applications.

Section 3

Delete temp files manually

Windows maintains two main temporary file locations that fill up over time. Both are safe to delete completely while no programs are actively running.

%TEMP% (user temp folder): Press Win + R, type %TEMP%, and press Enter. This opens your user-level temp folder, typically at C:\Users\[username]\AppData\Local\Temp. Select everything with Ctrl + A, then delete. You may get "file in use" errors for a handful of files — skip those and delete the rest.

Windows\Temp (system temp folder): Press Win + R, type C:\Windows\Temp, and press Enter. Delete the contents the same way. This folder typically requires administrator permission.

Close other applications first. Running programs write temporary files constantly. If you delete temp files while an application is writing to them, you may get errors or the application may crash. Close browsers, Office apps, and anything else before cleaning temp folders.

Between these two locations, it is common to recover 1–8 GB on a machine that has never had them cleared. De-Bloat Pro can automate this process, handling both folders and skipping in-use files automatically.

Section 4

Find and remove duplicate files

Duplicate files are one of the least obvious sources of wasted space because they're invisible — you don't know they exist until you go looking. They accumulate when you download the same attachment from email multiple times, copy a folder to back it up without remembering you already did, or migrate from an old PC and bring along a second copy of everything.

The only reliable way to find true duplicates is by comparing file contents, not names. Two files named photo_edited.jpg and photo_backup.jpg could be identical inside even though their names differ, and two files with the same name in different folders might be different versions of the same document.

File Deduplicator handles this by computing a SHA-256 hash of every file in your chosen folders and grouping files with matching hashes. The result is a list of verified duplicate groups with sizes, so you can see exactly what each deletion will recover before committing to anything.

Where to look first: Downloads folder, Desktop, Documents, and OneDrive/Google Drive sync folders tend to accumulate the most duplicates. Scanning all four in one pass typically finds the most wasted space.

Section 5

Hunt down large forgotten files

Sometimes disk space disappears not because of many small files but because of one or two very large ones you've forgotten about. Common culprits include old virtual machine disk images (.vmdk, .vhd), game install packages you downloaded and never deleted, video files from old projects, and database backups.

Windows has a built-in way to find these: open Settings → System → Storage → Other, which shows a breakdown by file type. For a more detailed view, the free tool WinDirStat visualizes your entire drive as a color-coded treemap, making it immediately obvious which files and folders are consuming the most space.

Once you've found the large files, verify you actually no longer need them before deleting. Virtual machine images and database backups in particular can look like junk but represent hours of work.

Section 6

Clear browser cache

Browsers cache every image, stylesheet, script, and video thumbnail you encounter while browsing. On a heavily used machine this cache can grow to several gigabytes over months of normal use. It exists to make page loads faster when you revisit sites, but it's completely safe to delete — the only side effect is slightly slower first loads on sites you visit frequently until the cache rebuilds.

Chrome and Edge: Press Ctrl + Shift + Delete, select "All time" from the time range dropdown, check "Cached images and files," and click Clear data.

Firefox: Press Ctrl + Shift + Delete, set the time range to "Everything," check "Cache," and click OK.

De-Bloat Pro can clear browser cache for Chrome, Edge, Firefox, and Brave in a single pass, including the browser-specific subdirectories that the in-browser clear sometimes misses.

Section 7

Clean up Windows Update leftovers

After Windows installs updates, it keeps a copy of the replaced system files in the C:\Windows\WinSxS folder. This allows you to uninstall recent updates if something breaks. The downside is that this folder can grow very large — on systems that have been running for several years without cleanup, WinSxS can reach 10–20 GB.

The safe way to clean this up is through Disk Cleanup. Press Win + S, search for "Disk Cleanup," right-click it and choose Run as administrator. Click Clean up system files, check "Windows Update Cleanup" and "Previous Windows installations" (if present), and click OK. This is safe to do once your current Windows version has been stable for a few weeks.

Note: After running Windows Update Cleanup, you will no longer be able to uninstall the updates that were cleaned. Only do this once you're confident the recent updates haven't caused any issues.

Section 8

Empty the Recycle Bin (and check its size limit)

This seems obvious, but it's easy to forget: files in the Recycle Bin still occupy disk space. If you've been deleting files for months without emptying it, the bin could be holding gigabytes. Right-click the Recycle Bin icon on the desktop and choose Empty Recycle Bin.

Less obvious: Windows sets a maximum size for the Recycle Bin per drive, and the default is often 10% of the drive capacity. On a 1 TB drive that's 100 GB — a significant amount of space that could be reserved for files you've already decided to delete. To reduce this, right-click the Recycle Bin, choose Properties, and lower the maximum size to something more reasonable like 4096 MB (4 GB).

Quick summary — space recovered per step:
Storage Sense: 1–5 GB • Temp files: 1–8 GB • Duplicate files: 2–20+ GB • Browser cache: 0.5–3 GB • Windows Update cleanup: 2–15 GB
