Section 1

What File Deduplicator does — and how SHA-256 works

Duplicate files accumulate quietly. You download the same PDF from email twice, drag photos from your phone onto your PC without checking what is already there, or back up a folder that itself contains a previous backup. Over time these copies consume gigabytes without any obvious sign they exist.

File Deduplicator finds these copies by comparing the actual contents of files, not their names. It uses SHA-256 hashing — a mathematical algorithm that produces a unique 64-character "fingerprint" for any given sequence of bytes. Two files with completely different names, in completely different folders, with different creation dates, will produce the same SHA-256 hash if and only if their contents are byte-for-byte identical. This makes it impossible to get false positives: if the app reports two files as duplicates, they genuinely contain identical data.

The scan runs in two phases. First, files are grouped by size — since two files of different sizes can never be identical, this step eliminates the vast majority of files without reading their contents at all. Only files that share an exact size are passed to the SHA-256 hashing phase. This size-first approach is what keeps scans fast even on drives with hundreds of thousands of files.

Why this matters vs. name-based finders

Many older duplicate finders compare file names or sizes only. Name-based comparison misses duplicates with different names (very common with downloads and camera imports). Size-based comparison produces false positives for files that happen to share a size. SHA-256 has neither problem.

Section 2

Downloading and first launch

Download FileDeduplicator.exe from the download page and save it anywhere — your Desktop or Downloads folder works fine. No installer is required. The application does not write to any system directories, does not create registry entries, and does not run background processes.

When you double-click the file for the first time, Windows SmartScreen may show a blue warning screen: "Windows protected your PC."

SmartScreen warning — this is expected

SmartScreen warns about any newly published executable that does not yet have a commercial code-signing certificate. This is not an indicator of malware. To proceed: click More info (small link below the warning text), then click Run anyway. You will only need to do this once — Windows remembers the decision. The full source code is publicly available on GitHub if you want to review it before running.

After the SmartScreen step (if it appeared), the application opens directly to its Configuration tab. There is no setup wizard, no "accept license" screen, and no installer that writes itself into Program Files.

Section 3

Adding folders to scan

The first step in the Configuration tab is selecting which folders to include in the scan. Click Add Folder to open a standard Windows folder picker. You can add as many folders as you want — the app combines them into a single scan.

The Include subfolders checkbox (enabled by default) causes the scanner to recurse into all sub-directories. Leave it enabled unless you only want to scan a specific folder's root level.

Where to start

These are the most productive folders to scan first, in order of how many duplicates they typically contain:

  • Downloads — the same installer downloaded multiple times, PDFs saved from email that already exist in Documents, images saved from a browser that are already in Pictures.
  • Pictures / Photos — phone syncs are a common source of exact duplicates, especially if you have used multiple sync tools over the years (iCloud, OneDrive, Google Photos backup, manual USB transfers).
  • Documents — version copies of reports, contracts, and spreadsheets ("Final_v2_FINAL_USE_THIS.docx").
  • Desktop — shortcuts and files accumulate here from multiple sources.
Avoid scanning system folders

Do not scan C:\Windows, C:\Program Files, or C:\Program Files (x86). System files that appear as duplicates are managed by Windows and should be left alone. Scanning these folders adds scan time without practical benefit.

Section 4

Choosing file type filters

The file type filter lets you narrow the scan to specific categories, making it faster and the results easier to review. If you are only trying to free up space from photos, selecting only Images means the scan skips every other file type entirely.

The 20 file types are organized into these categories:

  • Images — JPG, JPEG, PNG, GIF, BMP, TIFF, WEBP, HEIC
  • Videos — MP4, MKV, AVI, MOV, WMV, FLV, M4V
  • Audio — MP3, WAV, FLAC, AAC, OGG, M4A, WMA
  • Documents — PDF, DOCX, XLSX, PPTX, DOC, XLS, PPT, ODT
  • Archives — ZIP, RAR, 7Z, TAR, GZ
  • Executables — EXE, MSI, DLL
  • All files — scans every file regardless of extension
Best starting point for most users

Start with Images and Documents selected. These account for the majority of accidental duplicates on most PCs, and they are the easiest to verify before deletion. You can always run again with different filters.

Section 5

Picking your action

Before running the scan, choose what will happen to the duplicate files when you click Apply Action. There are three options:

1

Send to Recycle Bin (recommended for first-time users)

Duplicate files are moved to your Windows Recycle Bin. They are not permanently deleted — if you realise a file was needed, open the Recycle Bin and restore it. This is the safest option and the best choice if you are running File Deduplicator for the first time.

2

Move to folder

Duplicates are relocated to a folder you specify rather than deleted. This is useful if you want to review them manually — for example, move them to a temporary folder on your Desktop, look through them, then delete the ones you don't need. Good for large batches where you want full control.

3

Delete permanently

Files are removed immediately and cannot be recovered from the Recycle Bin. Only use this option when you are confident in the results — for example, after already running a "Move to folder" pass and reviewing the output. Not recommended for a first-time scan.

Section 6

Running the scan and reading progress

Once you have added your folders, selected your file type filters, and chosen your action, click Scan. The progress bar tracks two distinct phases:

Phase 1 — Size grouping (fast)

The app reads file metadata to group all files by size. This phase reads only directory entries, not file contents, so it typically completes in a few seconds even for folders with hundreds of thousands of files. At the end of this phase, the app knows exactly which files need to be hashed — and which can be skipped because no other file shares their size.

Phase 2 — SHA-256 hashing (disk-speed dependent)

For every group of size-matched files, the app reads their full contents and computes SHA-256 hashes. Speed depends on your storage hardware:

  • NVMe SSDs — typically 2–8 GB per minute processed
  • SATA SSDs — typically 1–4 GB per minute
  • HDDs — roughly 300 MB–1.2 GB per minute, depending on seek patterns

You can click Cancel at any time. The scan stops cleanly without leaving partial results or modifying any files.

The UI stays responsive during scanning

Scanning runs in a background thread. The application does not freeze, and you can cancel at any point without force-quitting.

Section 7

Understanding your results

Results are displayed as groups. Every file in a group shares the same SHA-256 hash — meaning they contain identical data. Each group shows:

  • Original (green label) — the file with the oldest modification timestamp in the group. The assumption is that the earliest copy is the "real" one. This is a useful heuristic, but it is not always correct — a file backed up to an external drive before the original was edited might appear as the "original."
  • Duplicate (amber label) — all other files in the group. These are pre-selected for the action you chose earlier.

Overriding which file to keep

You are not required to keep the file labelled "Original." Any file you uncheck is excluded from the action — only checked files are acted on. If you want to keep a file in a different location, uncheck it. You can also uncheck all files in a group to skip the group entirely.

Sorting the results

Sort by Size (largest first) to prioritise the groups where you will recover the most disk space. Sort by Group count (highest first) to tackle the files with the most copies first. The full path for every file is shown so you always know where each file lives on your drive.

Review before you act

Spend a minute scrolling through results before clicking Apply. Look for any file path that seems unexpected — an "Original" in an unusual location, or a group where you recognise all the copies and want to keep a specific one.

Section 8

Applying the action and the after-action report

Once you have reviewed results and confirmed your selections, click Apply Action. A confirmation dialog shows how many files are about to be affected and what action will be taken. Confirm to proceed.

File Deduplicator processes each selected file one at a time. A live log shows each file as it is processed. If any file cannot be acted on — for example, it is locked by another running application — the error is logged and the app continues to the next file. It does not stop or crash on errors.

The after-action report

When the action completes, a full summary report is generated showing:

  • Total files scanned
  • Duplicate groups found
  • Files acted on
  • Total disk space recovered
  • Any errors encountered (file locked, permission denied, etc.)

Click Save Report to export this as a .txt file. Keeping a record is useful if you want to review what was removed at a later date.

Section 9

Tips for best results

📥

Start with Downloads

The Downloads folder is the single highest-yield target on most PCs. The same installer, the same PDF, the same image — downloaded multiple times over months and years.

🗑️

Use Recycle Bin for your first run

Even if you are confident, the Recycle Bin option gives you a safety net. After you have reviewed a few successful scans, you can switch to permanent delete if you prefer.

📸

Target one folder type at a time

Scanning your entire C: drive at once produces a large result set that is harder to review. Run separate scans for Pictures, Documents, and Downloads to keep results focused.

🔁

Run it monthly

Duplicates accumulate gradually. A monthly scan of Downloads and Documents takes a few minutes and prevents space from quietly disappearing over time.

💾

SSD vs HDD speed expectations

On an HDD, large scans take longer. Run the scan when you are not actively using the PC, or before bed, so the hashing phase can finish without competing with other disk activity.

🔒

Close other apps before scanning

Applications that hold files open (Outlook, photo editors, video players) can cause "file locked" errors during the action phase. Closing them before you click Apply avoids these errors.
